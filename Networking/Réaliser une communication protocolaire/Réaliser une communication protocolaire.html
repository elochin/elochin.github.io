Réaliser une communication protocolaire

Chaque couche présentée précédemment implémente un protocole de communication qui permet à une couche donnée de l'émetteur de communiquer avec son équivalent au niveau récepteur. Par exemple, soit A un hôte établissant une communication avec B, le protocole IP de la couche IP de A communique avec son équivalent sur B, idem pour le protocole transport, liaison, applicatif, ...

Un protocole de communication est donc une spécification de plusieurs règles pour un type de communication spécifique. Par exemple, lorsque vous tapez une adresse du type http://www.isae.fr/ dans votre navigateur web, vous demandez explicitement une communication en utilisant le protocole HTTP vers la machine distante www.isae.fr. Ainsi, vous ouvrez un point de connexion ou socket, entre votre machine et la machine dénommée www.isae.fr en utilisant le protocole de communication HTTP qui offre un service de transfert hypertexte (i.e. service web).

Pour qu'une machine puisse communiquer avec une autre au travers du réseau, elle a donc besoin de son adresse ou de son nom et d'un port qui identifie le protocole de communication à utiliser. Ce port sera transformé en valeur numérique par la machine au même titre que son nom qui sera transformé en adresse IP numérique. Ces données sont essentielles à un programme pour renseigner la socket de communication que vous manipulerez par la suite.
