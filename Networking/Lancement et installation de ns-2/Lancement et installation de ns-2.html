Lancement et installation de ns-2

Depuis une salle info

Pour lancer ns-2 depuis une machine des salles info, ouvrez un terminal et tapez : module load ns-allinone/2.35 puis la commande : ns. A ce jour le simulateur est disponible uniquement en salles 61-001, 002, 014.

Sur votre PC Linux Debian/Ubuntu avec un paquetage

Pour installer ns-2 sous Debian ou Ubuntu, apt install ns2, il existe les paquetages suivants dans la distribution :

  • ns2 - Discrete event simulator targeted at networking research
  • ns2-dbg - Symboles de débogage de ns2
  • ns2-doc - Documentation pour ns2
  • ns2-examples - Exemples pour ns2

Enfin installer GNUPLOT via la commande apt install gnuplot-x11

Sur votre PC (autre système) avec les sources

Dans le cas où vous souhaiteriez compiler et installer ns-2 sur une autre distribution Linux (voire MacOSX ou Windows/Cygwin sans aucune garantie que cela fonctionne car je n'ai pas testé), téléchargez la distribution complète ns-allinone.

  • Décompressez l'archive avec tar xvzf ns-allinone-2.36.rc1.tgz
  • Puis cd ns-allinone-2.36.rc1/
  • Enfin ./install

A la fin de la compilation, regardez bien ce qui est indiqué dans le terminal, pour ma part :

Please put /home/elochin/ns-2/ns-2.36-elo/bin:/home/elochin/ns-2/ns-2.36-elo/tcl8.6.9/unix:/home/elochin/ns-2/ns-2.36-elo/tk8.6.9/unix
into your PATH environment; so that you'll be able to run itm/tclsh/wish/xgraph.

IMPORTANT NOTICES:

(1) You MUST put /home/elochin/ns-2/ns-2.36-elo/otcl-1.14, /home/elochin/ns-2/ns-2.36-elo/lib,
    into your LD_LIBRARY_PATH environment variable.
    If it complains about X libraries, add path to your X libraries
    into LD_LIBRARY_PATH.
    If you are using csh, you can set it like:
        setenv LD_LIBRARY_PATH <paths>
    If you are using sh, you can set it like:
        export LD_LIBRARY_PATH=<paths>

(2) You MUST put /home/elochin/ns-2/ns-2.36-elo/tcl8.6.9/library into your TCL_LIBRARY environmental
    variable. Otherwise ns/nam will complain during startup.

After these steps, you can now run the ns validation suite with cd ns-2.36.rc1; ./validate

For trouble shooting, please first read ns problems page http://www.isi.edu/nsnam/ns/ns-problems.html. Also search the ns mailing list archive for related posts.

Ne faire que les points #1 et #2 avec export LD_LIBRARY_PATH=/path... et export TCL_LIBRARY_PATH=/path... et les chemins (paths) qui vous serons retournés. Il vous faudra également installer le langage Perl et GNUPLOT.
